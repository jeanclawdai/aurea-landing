# AI Design Workflow (from @Av1dlive X thread)

## Core Philosophy
- **AI amplifies taste, not execution** — the skill is judgment, not prompting
- "AI can execute anything. You just need to know what's worth executing."

## The Workflow: Build Taste → Generate → Iterate

### 1. Build Taste Through Visual References
- Moodboarding works perfectly for vibe design
- Before AI, spend 20-30 min gathering screenshots of interfaces that solve the same problem
- **Where to look:** Awwwards (cutting-edge web), Pinterest AI search (upload screenshot → find similar), Dribbble (UI patterns)
- **Pro tip:** Ask "What specifically works here?" — hierarchy? spacing? color restraint? Name it to build vocabulary

### 2. Video → Code: The Kimi K2.5 Shortcut
- Record a video of any website → ask Kimi to "Clone this website with all the UX designs"
- Captures 80% of aesthetic immediately
- Parses structure, components, transitions frame-by-frame

### 3. Learn the Language of Design
**Typography:**
- Hierarchy (H1→H6→body→captions)
- Kerning (space between letters)
- Leading (vertical line spacing)
- Weight (thin/regular/bold/black)

**Layout:**
- White space (breathing room)
- Proximity (grouping related items)
- Contrast ratio (4.5:1 minimum for accessibility)

**Color:**
- Primary (brand, max 2)
- Accent (CTAs, highlights)
- Semantic (error red, success green, warning yellow)
- Neutral (text grays)

**Why:** When you say "reduce leading on H2s to 1.4 and tighten kerning by -0.02em," AI executes perfectly.

### 4. Meta-Prompts + Skills: Build Once, Use Forever
- **Meta-prompt:** Use AI to write prompts to feed to AI → produce dozens of "vibe specs" quickly
- **Skills:** Reusable task-scoped rule packs (constraints + checks + workflows)

**Top 3 skills mentioned:**
1. **UI Skills** (@Ibelick) — reusable fix packs for accessibility, motion, metadata
2. **UI/UX Pro Max** — searchable design playbook
3. **RAMS** — automated QA with line-level fixes

### 5. The Zoom-In Method: 50% → 99% → 100%
- **Pass 1 (50%):** Full context dump — goal, target audience, vibe, color palette, all pages/components, reference code
- **Pass 2 (99%):** Self-review — make AI reflect on its work (catches 70% of its own mistakes)
- **Pass 3 (100%):** Micro pass — pixel-perfect polish

## The New Design Moat
- **Precision:** Trade "make it look good" for professional terminology
- **Curation:** Use real-world references to kill generic outputs
- **Efficiency:** Let Zoom-In handle technical heavy lifting; focus on cohesion and empathy

## Bottom Line
"Stop overthinking and start shipping."
